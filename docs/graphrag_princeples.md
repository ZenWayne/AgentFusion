阶段一：build_index（离线索引构建）

整篇文档进去，知识图谱 + 向量索引出来。流水线按顺序执行 10 个 workflow：

输入: pd.DataFrame(id, text, title)  ← 我们传入的整篇 Markdown
                    │
                    ▼
┌──────────────────────────────────────────────────────────┐
│ 1. load_input_documents                                  │
│    读取 DataFrame，每行 → Document(id, title, text)       │
│    输出: documents.parquet                                │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 2. create_base_text_units                                │
│    Token-based chunking (默认 1200 tokens, overlap 100)   │
│    每个 chunk → TextUnit(id=sha512(text), document_id)   │
│    输出: text_units.parquet                               │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 3. create_final_documents                                │
│    反向映射: document → [text_unit_ids]                    │
│    输出: documents.parquet (enriched)                     │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 4. extract_graph  ★ 最耗时，大量 LLM 调用                  │
│    对每个 TextUnit 调用 LLM 抽取实体+关系:                  │
│    "从这段文本中提取所有实体(人/组织/概念...)及其关系"         │
│                                                          │
│    跨 TextUnit 合并:                                      │
│    - 同名实体 → 合并 description 列表 → LLM 摘要为一条      │
│    - 同源关系 → 合并 description + weight 累加              │
│    - 每个实体/关系保留 text_unit_ids (来源追踪)             │
│                                                          │
│    输出: entities.parquet, relationships.parquet           │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 5. finalize_graph                                        │
│    计算节点度数 (degree) 和边的 combined_degree            │
│    输出: entities/relationships.parquet (+ degree 列)     │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 6. extract_covariates (可选)                              │
│    LLM 提取声明/事实 (claims)，如果启用                     │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 7. create_communities  ★ 核心图算法                       │
│    Hierarchical Leiden 聚类算法:                           │
│    - 输入: 实体(节点) + 关系(边+权重)                      │
│    - 输出: 多层级社区结构 (level 0, 1, 2...)               │
│    - 每个社区记录: entity_ids, relationship_ids,           │
│      text_unit_ids, parent/children                      │
│                                                          │
│    输出: communities.parquet                              │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 8. create_final_text_units                               │
│    反向映射: text_unit → [entity_ids, relationship_ids]   │
│    输出: text_units.parquet (enriched)                    │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 9. create_community_reports  ★ 大量 LLM 调用              │
│    从底层社区到顶层，逐级生成摘要报告:                       │
│    - 输入: 社区内的实体描述 + 关系描述 + 下级社区摘要         │
│    - LLM 生成: title, summary, findings[], rating         │
│                                                          │
│    输出: community_reports.parquet                        │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ 10. generate_text_embeddings                             │
│     三组向量:                                             │
│     - text_unit.text → embedding                         │
│     - entity.description → embedding                     │
│     - community_report.full_content → embedding          │
│     存入 LanceDB 向量数据库                                │
└──────────────────────────────────────────────────────────┘

最终产出的 Parquet 文件

┌───────────────────────────┬──────────────────────────────────────────────────────────────────────────┬─────────────────────────────┐
│           文件            │                                  关键列                                  │            用途             │
├───────────────────────────┼──────────────────────────────────────────────────────────────────────────┼─────────────────────────────┤
│ documents.parquet         │ id, title, text, text_unit_ids                                           │ 原始文档                    │
├───────────────────────────┼──────────────────────────────────────────────────────────────────────────┼─────────────────────────────┤
│ text_units.parquet        │ id, text, document_id, entity_ids, relationship_ids, n_tokens            │ 文本块（GraphRAG 自动切的） │
├───────────────────────────┼──────────────────────────────────────────────────────────────────────────┼─────────────────────────────┤
│ entities.parquet          │ id, title, type, description, text_unit_ids, degree                      │ 抽取的实体                  │
├───────────────────────────┼──────────────────────────────────────────────────────────────────────────┼─────────────────────────────┤
│ relationships.parquet     │ id, source, target, description, weight, text_unit_ids                   │ 抽取的关系                  │
├───────────────────────────┼──────────────────────────────────────────────────────────────────────────┼─────────────────────────────┤
│ communities.parquet       │ id, level, entity_ids, relationship_ids, text_unit_ids, parent, children │ 社区层级                    │
├───────────────────────────┼──────────────────────────────────────────────────────────────────────────┼─────────────────────────────┤
│ community_reports.parquet │ id, title, summary, full_content, rating, findings, level                │ 社区摘要报告                │
└───────────────────────────┴──────────────────────────────────────────────────────────────────────────┴─────────────────────────────┘

完整关联链

Document (我们传入的整篇文章)
    │ text_unit_ids
    ▼
TextUnit (1200 token 的块, document_id → 回溯到文档)
    │ entity_ids, relationship_ids
    ▼
Entity (title, type, description, text_unit_ids → 回溯到文本块)
Relationship (source, target, description, text_unit_ids)
    │ community membership
    ▼
Community (entity_ids, text_unit_ids, level 层级)
    │
    ▼
CommunityReport (title, summary, findings, rating)

每一层都保留了向下追溯的 ID 列表，这就是 GraphRAG 的溯源基础。

阶段二：Search（在线查询）

LocalSearch 流程

用户 Query: "attention 机制的具体实现"
           │
           ▼
┌──────────────────────────────────────────────────────────┐
│ Step 1: 语义实体匹配                                      │
│   query embedding → 在 entity_description 向量库中检索     │
│   返回 top-k=10 个最相关的实体                             │
│   例: ["Multi-Head Attention", "Scaled Dot-Product",      │
│         "Self-Attention", "Query/Key/Value", ...]         │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ Step 2: 构建上下文窗口 (8000 tokens 默认)                  │
│   按比例分配 token 预算:                                   │
│                                                          │
│   ┌─ 25% Community Reports ──────────────────────┐       │
│   │  选中实体所属社区的摘要报告                      │       │
│   │  按 (命中实体数, 社区rank) 排序                  │       │
│   │  → context_data["reports"]                    │       │
│   └───────────────────────────────────────────────┘       │
│                                                          │
│   ┌─ 25% Local Context ──────────────────────────┐       │
│   │  Entities: 选中实体的描述+度数                   │       │
│   │  → context_data["entities"]                   │       │
│   │                                               │       │
│   │  Relationships: 实体间的关系                     │       │
│   │  优先级: 内部关系 > 互引外部关系 > 普通外部关系     │       │
│   │  → context_data["relationships"]              │       │
│   │                                               │       │
│   │  Covariates: claims (如果有)                    │       │
│   │  → context_data["claims"]                     │       │
│   └───────────────────────────────────────────────┘       │
│                                                          │
│   ┌─ 50% Text Units (原文片段) ──────────────────┐       │
│   │  选中实体关联的 text_units                      │       │
│   │  排序: 实体优先级 × 该 text_unit 中的关系数量     │       │
│   │  → context_data["sources"]  ← 溯源的关键!      │       │
│   └───────────────────────────────────────────────┘       │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ Step 3: LLM 生成回答                                      │
│   System Prompt:                                         │
│     "你是一个基于数据表格回答问题的助手...                    │
│      ---Data tables---                                   │
│      {community reports 表}                               │
│      {entities 表}                                        │
│      {relationships 表}                                   │
│      {sources/text_units 表}  ← 原文片段在这里             │
│      {claims 表}"                                         │
│   User Message: "attention 机制的具体实现"                  │
│                                                          │
│   → LLM 综合所有表格数据生成回答                            │
└──────────────────────┬───────────────────────────────────┘
                       ▼
返回 SearchResult:
  - response: "Attention 机制通过 Query/Key/Value 三个矩阵..."
  - context_data: {
      "reports": DataFrame,        # 社区报告
      "entities": DataFrame,       # 命中实体
      "relationships": DataFrame,  # 命中关系
      "sources": DataFrame,        # 原文片段 ← 溯源用这个
      "claims": DataFrame          # 声明 (可选)
    }

GlobalSearch 流程（对比）

用户 Query: "这些论文对 transformer 有什么共识？"
           │
           ▼
┌──────────────────────────────────────────────────────────┐
│ 准备: 按 community_level 筛选社区报告                       │
│   → 分成多个 batch (每 batch ≤ 8000 tokens)               │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ MAP 阶段 (并行):                                          │
│                                                          │
│ Batch 1 → LLM → [{answer: "要点A [Data: Reports(1,2)]",  │
│                    score: 85}, ...]                       │
│ Batch 2 → LLM → [{answer: "要点B [Data: Reports(3)]",    │
│                    score: 70}, ...]                       │
│ Batch 3 → LLM → [{answer: "要点C [Data: Reports(5,6)]",  │
│                    score: 60}, ...]                       │
│                                                          │
│ 提取 key points + importance score (0-100)                │
└──────────────────────┬───────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────┐
│ REDUCE 阶段:                                              │
│   过滤 score=0 → 按分数降序排列 → 截断到 token 预算         │
│   → LLM 综合所有 key points 生成最终答案                    │
│                                                          │
│   ⚠️ 只保留 Report ID 引用，TextUnit 级溯源丢失             │
└──────────────────────────────────────────────────────────┘

LocalSearch vs GlobalSearch 对比

┌─────────────────────────┬──────────────────────────────────────────┬────────────────────────────────────┐
│                         │               LocalSearch                │            GlobalSearch            │
├─────────────────────────┼──────────────────────────────────────────┼────────────────────────────────────┤
│ 入口                    │ query → 实体语义匹配                     │ query → 社区报告遍历               │
├─────────────────────────┼──────────────────────────────────────────┼────────────────────────────────────┤
│ 上下文来源              │ 实体 + 关系 + text_units + 社区报告      │ 只有社区报告                       │
├─────────────────────────┼──────────────────────────────────────────┼────────────────────────────────────┤
│ LLM 调用                │ 1 次                                     │ N+1 次（N个map + 1个reduce）       │
├─────────────────────────┼──────────────────────────────────────────┼────────────────────────────────────┤
│ 溯源精度                │ TextUnit 级（可追溯到原文片段 + 源文档） │ CommunityReport 级（丢失原文片段） │
├─────────────────────────┼──────────────────────────────────────────┼────────────────────────────────────┤
│ 适合场景                │ 具体问题："attention 怎么实现的？"       │ 宏观问题："整体有什么共识？"       │
├─────────────────────────┼──────────────────────────────────────────┼────────────────────────────────────┤
│ context_data["sources"] │ 有（TextUnit DataFrame）                 │ 无                                 │
└─────────────────────────┴──────────────────────────────────────────┴────────────────────────────────────┘
